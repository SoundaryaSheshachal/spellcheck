Spell Checker Tool – User Guide

1. Introduction

The Spell Checker tool automates the task of spelling-check of documents saved in text files (plain text or in MS Word format). This helps the users to avoid going though the whole document page by page to check the spelling errors. Even with MS Word that has inbuilt spell checkers, user has to walk through all the errors one by one or page by page -you cannot just see an “errors report”. This tool lets the user see all misspells as a list and decide what to do with them.

2. How it works

This tool first initializes its inbuilt English dictionaries and any custom dictionaries that the user may have selected. After that when the user supplies it an input file to check the spellings, this tool does the following:

  1. Extract ALL the text found in the input document.
  2. Breaks the text into words. A word is any contiguous sequence of characters that doesn’t contain a space, hyphen or a punctuation mark.
  3. For each word it then tries to:
    1. Check if it is a number or abbreviation etc., if yes then it is ignored.  
      Else:
    2. Find if any dictionary contains the word. If not then the word is added into the list of misspelt words. Search in the dictionaries is case insensitive. 

3. Features

  1. Supports plain text and MS Word document.
  2. Allows saving the spelling errors report to a file.
  3. Includes both US and UK English dictionaries by default.
  4. User can supply multiple custom dictionaries.
  5. Allows the user to selectively add misspelt words to custom dictionary.
  6. Settings for a user are persisted i.e. the user will find the same settings on next usage of the program.

4. Using the tool

Normally the user will do the following sequence of steps for utilizing this tool:

  1. Select the input file by clicking on the “Select input file” button. It only allows the *.txt and *.doc file to be selected.
  2. Check the spellings by clicking on “Check spellings” button. This may take from few seconds to several minutes depending on the size of the input file.
  3. Spelling errors if any found in the input file will be shown to the user in the “Spelling errors” text pane. Count of errors is displayed in the bottom left corner.
  4. At this point the user may do one or more of the following:
    1. Save the errors in a file by clicking on “Save results as”. This errors file can be used to create a dictionary. For example, in the very first run of the tool on a book that contains non-English words, all such words will be listed as errors. User may save the results as a file (can be named anything), then keep only the valid non-English words in it and add the file as a custom dictionary (explained below).
    2. Add some of the words from errors list to the custom dictionary. This can be done by highlighting the word (or multiple words) and then clicking on the “Add selected to dictionary” button.
    3. After adding the words as above user may re-run the spelling check on the same document.

5. Custom dictionaries

This tool allows the user to add any number of custom dictionaries against which to check the spellings. A custom dictionary file is a text file containing one word per line. 

Dictionary settings can be changed by clicking the “Dictionary settings” button. This button opens a dialog box that has two sections:

  1. Dictionaries section: Here the user can “Add” or “Remove” custom dictionaries and also set one of them as a “Default” custom dictionary. A default custom dictionary is the one where new words get added when user adds the highlighted errors via “Add selected to dictionary” button on the main window.
  2. Miscellaneous section: At present there’s only one setting in this section that allows the user to set whether the upper case words are to be ignored while checking the spellings. You can also change the word delimiters here. Unicode values for different delimiters should be specified. See here for a list of unicode values: http://www.unicode.org/charts/
